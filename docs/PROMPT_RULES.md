# PROMPT RULES

## ALWAYS DO

- reference specific file or feature
- keep prompts focused
- build one feature at a time

---

## NEVER SAY

❌ "build the whole app"
❌ "add everything"
❌ "make it advanced"

---

## GOOD PROMPTS

- "Create a SystemList screen using dynamic data (no hardcoding)"
- "Build a reusable CaseCard component"
- "Set up Firebase service for fetching cases"

---

## WHEN CURSOR DRIFTS

Say:

"Follow PROJECT_RULES.md and ROADMAP.md strictly. Remove unnecessary code."

---

## WHEN UNSURE

Ask:

"What is the minimal implementation for this?"
